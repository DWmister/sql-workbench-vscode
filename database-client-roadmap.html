产品决策原则 Decision Rules
  1. 1
    先做窄，但要闭环

    v0.1 聚焦 MySQL / PostgreSQL / SQLite，先完成连接、Schema、SQL 和只读结果闭环；查询历史后移。

  2. 2
    免费替代 Premium 高频点

    v0.2 已交付 CSV / JSON / XLSX、JSON 结果、Workspace 连接、完整表 DDL、变量与 CodeLens。

  3. 3
    所有写入都要看得见

    结果网格和表属性永久只读。修改数据或结构必须通过用户可审阅、可执行的 SQL。

  4. 4
    ER 与 Service 面板延后

    当前优先增强 SQL、结果、导出和表属性；ER 图进入 v1.5，Redis、SSH、Docker 等能力进入长期路线。

v0.1已交付

MVP 查询闭环

让用户可以卸载普通 SQL 查询插件，用我们完成日常查库工作。

MySQL PostgreSQL SQLite
  • Activity Bar + Database TreeView
  • 连接新增、测试、删除、分组、本地持久化、无限连接
  • SecretStorage 保存密码，连接树展示表与列
  • 点击 Table 在当前编辑栏显示只读列信息
  • 结果表格与列信息只读，所有修改必须通过 SQL
  • SQL 写在 VS Code 原生 .sql 编辑器中
  • 状态栏显示并切换当前连接，QuickPick 标注 Current
  • 执行当前语句 / 选区 / 全部语句
  • 结果 Webview：分页、错误、耗时、影响行数
  • 基础 SQL 提示：关键字、片段、表名、字段名

Connection + Driver

推荐架构图

核心是 Extension Host 中的能力编排，Driver 适配层和 Webview UI 解耦。数据、密钥和执行链路都要可审计。

VS Code Contributions Activity Bar、TreeView、Commands、SQL Language、Snippets、Settings。
Connection Store globalState / workspace config 保存非敏感字段，SecretStorage 保存密码、token、私钥。
Safety Layer 危险 SQL 确认、生产连接标识、AI 执行确认、查询日志。
Extension Host Service Manager · Driver Registry · Query Runner · Metadata Cache
Driver Adapters mysql2、pg、SQLite、tedious、ioredis、mongodb、ssh2，以统一接口暴露能力。
Webview Apps Result Viewer、Export Dialog、Table Properties、Connection Form，以及未来的 DDL / Mock SQL Generator 和 ER Diagram。
External Tools mysqldump、pg_dump、docker、ssh tunnel、用户自有 S3 / WebDAV 同步。
