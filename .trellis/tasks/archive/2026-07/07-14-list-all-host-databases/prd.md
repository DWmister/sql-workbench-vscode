# 补充：连接展示主机全部数据库

## Goal

让一个 MySQL/MariaDB 或 PostgreSQL 主机连接代表“主机与账号”，而不是只能代表配置时填写的单个数据库。用户可以不填写默认数据库，连接后从活动栏数据库树浏览当前账号可访问的全部数据库。

## Background

- 原连接表单把数据库设为必填项，保存后数据库树只展示该连接对应的一组 Tables。
- 竞品的主机连接节点可以继续展开数据库列表，用户期望 SQL Workbench 具备同样的浏览能力。
- 本任务是对已经完成并处于暂存区的 `0.2.4` 功能补录 Trellis 需求与验收资料，不重复改写现有实现。

## Requirements

### R1. 服务端数据库改为可选

- MySQL/MariaDB 与 PostgreSQL 连接表单中的数据库字段必须标记为“默认数据库（可选）”。
- 保存和测试服务端连接时，不得因为数据库为空而阻止操作。
- 已填写的数据库仍作为直接执行 SQL 与补全时使用的默认数据库，兼容已有连接配置。
- SQLite 仍必须填写数据库文件路径，不受本次改动影响。

### R2. 主机连接展示可访问的全部数据库

- 展开 MySQL/MariaDB 连接时，通过当前连接凭据加载账号可见的数据库。
- 展开 PostgreSQL 连接时，只加载非模板且当前账号具有 `CONNECT` 权限的数据库。
- 数据库名称按名称排序展示。
- 加载失败时在对应连接下展示错误信息，不影响其他连接节点。

### R3. 数据库节点携带独立上下文

- 展开某个数据库后，Tables、表、列及表属性读取必须使用该数据库作为连接上下文。
- 同一个主机下不同数据库生成不同的树节点标识，避免 VS Code 将节点状态错误复用。
- 展开数据库只派生运行时连接上下文，不得为每个数据库新增或覆盖持久化连接配置。

### R4. 文档与版本说明同步

- README 中说明服务端默认数据库可留空，以及数据库树会列出当前账号可访问的全部数据库。
- CHANGELOG 和扩展版本记录该能力。

## Acceptance Criteria

- [x] AC1：新增或编辑 MySQL/PostgreSQL 连接时，数据库为空可以通过表单校验、测试并保存。
- [x] AC2：数据库字段显示为“默认数据库（可选）”，且不带必填标记。
- [x] AC3：展开 MySQL/MariaDB 主机连接后，显示 `SHOW DATABASES` 返回的可见数据库。
- [x] AC4：展开 PostgreSQL 主机连接后，显示非模板且具有 `CONNECT` 权限的数据库。
- [x] AC5：展开任一数据库后，Tables 与后续表/列加载使用该数据库上下文。
- [x] AC6：同主机下不同数据库的 Tables、表和列节点 ID 相互隔离。
- [x] AC7：SQLite 连接仍直接展示 Tables，并继续要求数据库文件路径。
- [x] AC8：自动验证覆盖表单可选状态、数据库枚举和数据库上下文传递。
- [x] AC9：中英文 README、CHANGELOG 与包版本已同步到 `0.2.4`。

## Out of Scope

- 展示 Views、Functions、Procedures 等更多数据库对象。
- 修改数据库权限或展示当前账号无权连接的数据库。
- 把每个数据库保存为单独的连接配置。
- 改变 SQL 结果只读、表属性只读等既有产品边界。
