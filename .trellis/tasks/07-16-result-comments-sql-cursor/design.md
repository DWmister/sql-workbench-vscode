# 技术设计：结果字段注释与 SQL 光标执行边界

## 范围

本任务包含两个互不耦合但同属 SQL 使用体验的改动：查询执行层补充结果字段注释元数据，解析层修正两条语句间纯空白的光标归属。

## 结果字段注释数据流

```text
数据库驱动执行 SQL
  -> 驱动返回结果字段来源标识；兼容协议缺失时解析简单 SQL 投影来源
  -> 批量查询数据库系统目录中的字段注释
  -> QueryColumn { name, type, comment? }
  -> ResultViewPanel payload
  -> 表头第一行字段名、第二行注释、title 完整提示
```

### 共享结果契约

在 `src/results/types.ts` 的 `QueryColumn` 增加 `comment?: string`。字段可选以保持 SQLite、表达式列、无权限读取注释和旧数据路径兼容。导出逻辑继续只读取 `column.name`。

### MySQL/MariaDB

标准 `mysql2` 的真实运行时字段来源优先使用 `(schema || db, orgTable, orgName)`：`schema` 是运行时主字段，`db` 仅保留兼容回退。仅对可识别的来源三元组去重并批量查询 `information_schema.columns.column_comment`，再按来源回填结果列。

- 别名列若仍有可靠 `orgName`，可以显示源字段注释。
- MySQL 协议兼容数据库（实际为 StarRocks）可能把 `schema`、`orgTable`、`orgName` 都返回为空。此时只解析简单 `SELECT` 投影：读取 `FROM` / `JOIN` 的表别名，将合格的 `别名.字段` 按结果列位置映射到来源字段。
- 位置是回退映射的一部分：`profile.document_type` 与 `verification.document_type` 即使结果标签相同，也必须保留不同的来源表。
- 单表 `SELECT *` 或 `别名.*` 可将返回字段名映射到唯一已解析表；多表通配符、表达式、投影数量与返回字段数不一致、无法识别的别名均不映射。
- `DISTINCT`、`DISTINCTROW` 和 `ALL` 是简单字段投影的选择修饰符，来源解析在匹配 `别名.字段` 前剥离它们；这不扩展到函数、计算表达式或其他复杂投影。
- 计算表达式缺少可靠来源时不显示注释。
- 元数据查询复用当前数据库连接和查询超时。

### PostgreSQL

`pg` 的 Field 提供 `tableID` 和 `columnID`。仅对 `tableID > 0` 且 `columnID > 0` 的字段去重，批量查询 `pg_description`，按表 OID 与列序号回填注释。

- 表字段别名仍保留来源 OID/列号，可显示注释。
- 表达式字段通常返回 0，不显示注释。
- 元数据查询复用当前 Client。

### 失败降级与分页

注释属于辅助元数据。系统目录查询失败时捕获错误并返回无注释列，原 SQL 的 rows、rowCount、分页和错误状态不得改变。

首页、后续分页和全量导出都经过相同的列构造函数，避免翻页后注释消失。当前任务不增加持久化缓存，优先保证结果一致性；只有存在标准驱动来源或已解析的兼容协议来源时才发送元数据查询。来源无法解析时记录诊断警告，但不向结果面板返回错误。

### 表头展示

结果表格表头拆成两个 span：字段名始终显示；非空注释以弱化颜色、小字号第二行显示，单行截断。`th.title` 组合完整字段名和注释供悬停查看。无注释字段不渲染第二行。

## SQL 光标边界

`scanStatementRanges` 继续负责识别字符串、标识符、注释和真实分号，不修改拆句规则。

只调整 `findStatementAtOffset` 的间隔回退策略：

1. 光标在某个语句范围内时返回该语句。
2. 光标位于第一条语句之前时返回第一条。
3. 光标位于两条范围之间时返回前一条，不再寻找下一条。
4. 光标位于最后一条之后时返回最后一条。

下一条语句的前置注释已经包含在下一条 range 内，因此光标进入注释时仍返回下一条；只有 range 之间的纯空白归前一条。

## 兼容性与回滚

- `QueryColumn.comment` 可选，不改变现有消费者。
- 选区执行和执行全部不经过新的间隔选择分支，行为不变。
- 导出格式不增加注释列或修改表头名称。
- 可分别回退列注释解析和 `findStatementAtOffset` 策略，不涉及数据迁移。
