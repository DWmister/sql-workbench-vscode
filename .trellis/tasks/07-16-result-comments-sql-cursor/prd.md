# 优化结果字段注释与 SQL 光标执行边界

## Goal

提升日常 SQL 查看与快捷执行体验：结果表格能直接看到真实字段的数据库注释；光标位于两段 SQL 之间时，快捷键执行符合“刚写完上一条 SQL”的直觉。

## Background

- `QueryColumn` 当前只有 `name` 和 `type`，结果表头只显示字段名。
- MySQL 驱动真实运行时结果字段使用 `schema`、`orgTable`、`orgName`，类型声明还保留可选 `db`；实现必须以 `schema` 为主并兼容 `db`，用于批量查询 `information_schema.columns.column_comment`。
- MySQL 协议兼容数据库（实际验证的 StarRocks）可能能够查询 `information_schema`，但结果字段不返回原始表/字段标识；不能因缺少该元数据而直接跳过全部字段注释。
- MySQL `information_schema` 查询必须为返回字段显式设置稳定别名，不能依赖服务端返回键的大小写。
- PostgreSQL 驱动结果字段包含 `tableID`、`columnID`，可用于批量查询 `pg_description`。
- SQLite 和计算表达式/无法关联来源字段的别名没有可靠字段注释来源，应保持无注释而不是猜测。
- `findStatementAtOffset` 当前在光标不属于任何语句范围时优先查找下一条语句，导致分号后的空白区域执行后面的 SQL。

## Requirements

### R1. 结果列携带字段注释

- `QueryColumn` 增加可选注释字段，且不影响现有 name/type 使用方和导出字段名。
- MySQL/MariaDB 对可识别真实来源的结果列加载字段注释。
- 对缺少驱动来源元数据的 MySQL 协议兼容数据库，简单字段投影应仍可显示注释：按 SQL 中的表别名和结果列位置关联来源，避免同名结果列被错误地映射到同一字段。
- PostgreSQL 对可识别真实来源的结果列加载字段注释。
- 字段注释查询失败不得让原 SQL 执行失败；结果仍应正常显示，只是不展示注释。
- 分页首页、后续页和导出全部数据返回的列元数据行为保持一致。
- SQLite、计算表达式和无可靠来源的列保持无注释。
- 单表 `SELECT *` 或 `别名.*` 可使用返回字段名关联该表；多表通配符、表达式、投影数量不匹配或其他歧义场景保持无注释，不能仅按显示列名猜测。
- 简单字段投影前的 `DISTINCT`、`DISTINCTROW` 或 `ALL` 选择修饰符不应阻断来源关联。

### R2. 结果表头展示字段注释

- 仅在 comment 非空时展示，不能为无注释字段占用额外内容。
- 表头的字段名仍是主信息，注释是辅助信息。
- 表头采用两行展示：第一行字段名，第二行以弱化颜色显示字段注释。
- 过长注释在表头省略，鼠标悬停显示完整的字段名和注释。
- CSV、JSON、XLSX 导出的列名不拼接注释。

### R3. 光标位于语句间隔时执行前一条 SQL

- 未选择文本时，光标从上一条 SQL 的分号位置开始，直到下一条 SQL 的首个有效字符之前，都应解析为上一条 SQL。
- 两条 SQL 之间的纯空白归前一条；下一条 SQL 的前置注释归下一条，保持注释说明后续语句的既有语义。
- 光标落到下一条 SQL 的首个有效字符或其正文中时，执行下一条 SQL。
- 文档开头到第一条 SQL 之间仍解析为第一条 SQL；最后一条 SQL 之后仍解析为最后一条 SQL。
- 字符串、标识符和注释内部的分号仍不得拆分语句。
- 显式选中文本与“执行全部”行为不变。

### R4. 版本与 VSIX 发布收尾

- 上一批产品改动已经提交后，开始新修改时才更新一次扩展版本，并同步锁文件、README 版本徽章和 CHANGELOG。
- 当前产品改动尚未提交时，后续修复不得再次增加版本号。
- 每次修改并验证通过后都必须重新生成与当前版本一致的最新 VSIX，即使版本号不变也要覆盖重建。
- 根目录 `AGENTS.md` 必须固化上述规则，并明确本地 `s.sql` 不得提交。

### R5. 路线与开源发布校验

- 本次结果效率增强需与 `0.2.x` 路线图对齐，并同步中英文 README 与路线图看板。
- 每次更新在实现前和发布前都必须进行路线校验；更适合下一中版本或后续版本的能力，须先向用户说明并取得确认。
- 对开源仓库暂存内容执行敏感信息审查；真实凭据、私有地址、连接串和内部数据库标识不得推送。

## Out of Scope

- 为 SQLite 推断或解析非标准字段注释。
- 为计算表达式、聚合结果或无法确定来源的别名猜测注释。
- 为多表通配符、投影数量不匹配或其他歧义 SQL 猜测字段来源。
- 将注释拼进导出文件的字段名。
